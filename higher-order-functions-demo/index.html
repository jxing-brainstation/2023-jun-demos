Higher-order function: A function that returns a function or takes other functions as arguments.

Higher Order Array Methods From the slides:

Array.prototype.forEach()

Parameters: A callback function that is called with 3 arguments: the element, the index, and the array itself. This function should not return anything.

Return value: undefined

Array.prototype.map()

Parameters: A callback function that is called with 3 arguments: the element, the index, and the array itself. The function should return the new element to add to the new array.

Return value: a NEW ARRAY with each element changed.

Array.prototype.filter()

Parameters: A callback function that is called with 3 arguments: the element, the index, and the array itself. The function should return either true or false to indicate whether or not you wish to keep the value.

Return value: A shallow copy of the original array that is filtered or an empty array.

Array.prototype.find()

Parameters: A callback function that is called with 3 arguments: the element, the index, and the array itself. The function should return either true or false to indicate whether or not a matching element is found.

Return value: the first element in the array that satisfies the testing function or undefined if nothing is found.

Array.prototype.sort()

Parameters: An optional function that defines the sort order. This function is called with two arguments: a and b which represents the first and second elements that are being compared. This function should return an integer. If the integer is positive, a will be sorted after b. If the integer is negative, a will be sorted before b. If the integer is 0, the original order of a and b will be kept.

Return value: a reference to the original array that is now sorted.

NOTE: this will mutate the array itself.
